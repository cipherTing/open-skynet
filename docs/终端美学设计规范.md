# 数字赛博格终端美学（Digital Cyborg Terminal Aesthetic）设计规范

> 本规范是 Skynet 首页（欢迎页）视觉与交互系统的最高纲领。
> 定位：生物黑客终端（Bio-punk CLI）× 未来硬核科幻控制台。将冰冷严谨的数字化工业框架与无序流动的生命体数据流进行视觉与交互层面的融合。
> 禁止向传统现代商业网页美学妥协或平庸化处理。

## 一、严苛的色彩系统与光度学控制规范（Color & Photometry Specification）

**绝对暗域底色基调（Absolute Dark Base）**：画布与容器的底层背景必须是纯粹、深邃且无底的绝对黑色（十六进制代码限定为 `#000000`）。只允许在极少数次要背景层中，融入明度极低、近乎不可察觉的微弱暗绿调（如 `#040704`）。严禁使用任何现代网页常见的柔和灰色渐变、商务蓝调、深蓝灰或带有毛玻璃光晕的彩色环境微光，背景必须像虚无的深空或完全关闭的阴极射线管（CRT）屏幕般冰冷、无情。

**高饱和度强光级激活色（Radiant Energy Accent）**：系统的视觉核心能量色、高亮提示色与交互反馈色，必须是具有极强电子发光感、放射感与生物毒性的荧光黄绿色（如 `#ADFF2F` 或 `#A7FC00`）。这种颜色是界面的"灵魂流动"，专门用于承载系统的最高亮数据、核心扫描线、动态视准线以及关键的系统激活状态。它在纯黑背景上的对比度必须达到刺眼的阈值，形成强烈的视觉压迫感。

**高对比度权威文本色（Authoritative Text）**：常规核心文本、大字标题和高级别数据流必须使用绝对的纯白色（`#FFFFFF`）或明度在 95% 以上的冷白色。它不带有任何情感色彩，仅代表了系统输出的最高权威性、客观性与绝对的可读性，与暗色背景形成刀锋般锐利的对比。

**低明度系统噪音色（Background Noise Color）**：所有网格线、非激活状态的坐标刻度、次要的遥测数据标签以及背景装饰线条，必须统一使用低明度的暗绿色（如 `#1A2E1A` 或 `#122012`）。它们作为页面的视觉背景噪声存在，甘愿退居底层，在视觉上形成丰富的暗部细节，从而衬托出激活色与文本色的锐利感。

## 二、字体层级对抗与宏微观排版力学（Typography & Layout Dynamics）

**巨型无衬线标题宏观排版（Macro-Typography）**：页面中最高层级的视觉锚点，必须由极具分量感、纯几何形态、字重极大的粗无衬线体（如 Impact、Helvetica Neue Black，或大字重的变形几何体）构成。这些巨型标题的字间距（Tracking）必须进行极限收紧，甚至故意让字母边缘产生近乎触碰的几何压迫感；其行高（Leading）同样必须压缩至极限（如 `line-height: 0.85`）。它们不以传统的舒适易读性为第一目标，而是作为一种充满力量感的视觉几何块面，横亘在画布上，传达不可动摇的系统宣告性与冷酷的科技美感。

**微型等宽遥测文本微观排版（Micro-Typography）**：与巨型标题形成极端对立的是，界面中所有的次要标签、运行参数、十六进制代码、经纬度坐标、伪随机状态码和实时日志流，必须强制使用高精度、绝对等宽的科技字体（Monospace，如 JetBrains Mono、Courier New 或 SF Mono）。其字号必须控制在人类视觉可读的边缘（等效像素 10px - 12px）。与大标题相反，微型文本的字间距必须大范围加宽（如 `letter-spacing: 0.15em`），呈现出精密物理仪器印刷或古老终端代码输出的工业质感与间歇性节奏。

**非对称式空间构图与绝对刚性边界**：彻底抛弃传统的中心对称、黄金分割或三段式平庸布局。采用极端的四角包围式或非对称重心排版。页面的边缘与角落应当填满高度密集的系统元数据（如世界时区、数据校验和、协议版本、在线状态指示器），形成一个封闭的精密控制台框架，而画布中心则留出用于数据解析的大面积负空间。废除所有带圆角（`border-radius: 0`）的容器、以及大面积的纯色底色块。所有区域模块必须是直角矩形，其边界由 1px 的固体细线或者由绝对定位的四个角标元素（利用伪元素构建的 L 型直角边框）来精确界定。

## 三、图形符号化与数字机理构成（Graphics & Texture Constituents）

**字符化矩阵肌理（ASCII/Matrix Texture）**：严禁使用任何常规的平滑几何图表、现代线性扁平插画或真实拟物照片。界面中的核心视觉图形（如抽象的生物轮廓、复合数据包模型、空间拓扑结构等）必须完全由高低明暗不同的 ASCII 字符、数字、标点符号（如 `@`、`#`、`S`、`+`、`=`、`:`、`.`、`1`、`0`）组成的"字符点阵矩阵"来渲染。通过字符排布的疏密程度来模拟传统图形的明暗、体积和阴影，制造出一种图像正在被系统底层代码实时解构、扫描并数字化的粗粝像素机理。

**工业平视显示器（HUD）视准元件**：画布中应大量充斥宽度仅为 1px 的精致几何线框。包括但不限于：细线十字准星、四角 L 型定位边框、带有度数刻度的旋转环、无间断的百分比滑块条、条形码矩阵以及实时的脉冲波形图。这些元件不带有任何现代网页常用的发光投影或羽化边缘，必须保持绝对的物理精确度与矢量锐利度。

**点阵栅格化矩阵（Grid Matrix）**：整个画布的底层或特定信息模块内部，必须隐约可见一层由 1px 的点状网格（Dot Grid）或极细微弱暗绿色线构成的栅格系统。所有 UI 元素的边缘、文本的起始线、线框的交汇点，必须完美地与栅格交点对齐，不得出现任何无理的随意错位或浮动像素。

## 四、动态遥测系统与高频状态机（Dynamic Telemetry & Live State Machines）

**非周期性毫秒级数值跳动（Data Micro-Fluctuations）**：静态是这个风格的敌人。页面中所有渲染数字、百分比、坐标参数的节点，决不能保持静止。必须通过高频数据状态机驱动，使这些数值在极小的合理范围内进行高频跳动（例如：一个系统效率参数在 99.942% 到 99.978% 之间以 150ms - 300ms 的随机间隔进行微幅摆动）。这种摆动必须是非周期性的随机数学模拟，以此营造出系统正在进行不间断、高灵敏度实时监测与运算的鲜活生命力。

**画布级字符矩阵滚动引擎（Canvas ASCII Stream）**：针对图像区域或主视觉背景，必须利用高效渲染机制（如 Canvas 渲染管线）构建高密度的字符点阵渲染引擎。依据图像的灰度矩阵，将实时的字符流动态绘制在画布上。且这些字符必须伴随着微弱的垂直流向运动或规律性的随机闪烁更新，以保持视觉底层的动态机理，同时通过离屏渲染等手段确保底层性能损耗降到最低。

**滚动式系统日志总线（System Log Stream）**：在页面的特定边缘面板中，建立一个无休止的日志流组件。该组件高频接收伪系统事件字符串，新日志从底部推入，旧日志向上滚动并淡出消失，滚动过程必须是清脆的逐行硬切换（Line-by-line snap），严禁出现平滑、顺畅的阻尼滚动动效。

## 五、微交互、光标靶向与乱码矩阵机制（Micro-interactions & Scramble Text）

**字符级乱码变换解码动效（Text Scramble Effect）**：当用户鼠标悬停（Hover）在任意可交互元素（如导航标签、功能开关、控制锚点）上时，目标文本不能直接改变颜色或产生渐变，而是必须触发一个短暂的"数据重组解码动效"：在 200 毫秒的时间窗口内，文本内的每个字符会高速随机变换为其他 ASCII 符号或二进制数字，随后如同密码破译般，从左至右逐个固定为最终的正确文本，并伴随短暂的高亮。

**光标靶向视准跟随系统（Targeting Custom Cursor）**：隐藏原生鼠标光标，使用前端自定义的 DOM 节点构建一个高精度 HUD 光标。该光标由一个 1px 的细线十字准星以及一个动态跟随的直角边框方框组成，并在光标下方实时用等宽微型字体显示当前的屏幕绝对 X/Y 像素坐标以及相对画布的伪经纬度变换参数，使用户每一次移动鼠标都感觉是在操控一台精密雷达。

**触发式屏幕震撼与干扰反馈（Glitch & Screen Shake）**：当用户点击核心控制行为或触发重大状态切换时，整个页面（或目标模块）必须短暂触发一个硬核的故障反馈动效：通过矩阵变换在 100 毫秒内进行高频微幅移位（模拟受到物理冲击时的屏幕震动），同时叠加一个通过剪裁路径（clip-path）纵向切割界面的瞬间错位闪烁（Glitch Offset），随后立即恢复正常，给予交互以强烈的工业能量反馈和机械碰撞感。

## 六、空间过渡、硬切与步骤渲染算法（Spatial Transition & Step Animations）

**彻底摒弃平滑缓动曲线（Anti-Blurry Interpolation）**：在定义所有动画、过渡与转换时，严禁使用 ease、ease-in-out 或 linear-bezier 等会让视觉产生连续模糊过渡的平滑曲线。所有位置移动、尺寸伸缩、显示隐藏，必须全部使用 `steps()` 步进函数（如 `transition-timing-function: steps(4, end);`）。动效应当像机械齿轮般一格一格地跳跃，或者像电子束扫描般瞬间突变，消除任何软绵绵的现代商业感。

**水平扫描线转场机制（Scanline Transition）**：当切换大模块或刷新数据视图时，目标区域必须自上而下扫过一条高亮为荧光黄绿色、厚度为 2px 的水平扫描线。线流经之处，底层的 UI 元素和数据才如同被高能束打印般逐行显现，创造出强烈的系统扫描解构体验。

**性能与渲染管线防御（Performance Optimization Rules）**：所有的动态微交互（光标跟随、数值跳动、ASCII 流动）必须进行严格的性能防御，确保在 60fps 下稳定运行。频繁改变的样式必须使用 GPU 加速（如 `transform: translateZ(0)`）。避免引起浏览器的重排（Reflow），所有动态更新的数据节点必须固定长宽与行高，防止文本跳动导致周围布局发生无理的崩塌或整体位移。

## 实施修订记录（与原始规范的偏差，按决策时间登记）

> 以下为首页实现过程中确认的工程决策，与本规范正文具有同等效力；冲突时以本节为准。

1. **光标跟随策略（2026-07 修订）**：原始规范中外围方框采用 lerp 惯性延迟跟随，实测手感拖沓。现行实现为**十字准星、方框、坐标读数全部 100% 跟手**（`pointermove` 直接写入位置），零插值、零追赶；方框仅在 hover 可交互元素时保留 120ms `steps()` 尺寸/颜色切换。
2. **弹窗场景光标回退（2026-07 新增）**：页面出现模态层（`[role="dialog"]` / `[aria-modal="true"]` 等）时，自定义光标必须完全隐藏并恢复系统原生光标；弹窗关闭后自动恢复。下拉菜单、Tooltip、Toast 不触发回退。
3. **连续旋转豁免（2026-07 新增）**：`steps()` 步进为原则，但雷达扫掠等以"流畅运转"为核心观感的元件（如接入协议区块的雷达表盘）允许 rAF 时间驱动的**连续平滑旋转**；是否豁免由观感评审决定，逐案登记。
4. **主题策略（2026-07 确认）**：首页不提供明暗模式切换，强制绝对暗域；页面颜色硬编码，不依赖全站主题变量。
5. **文案基调（2026-07 确认）**：终端冷峻感与人味并存——区块标题一律中文（英文代号仅作装饰性微型文本）；禁止「不是……而是……」等对仗否定句式与空洞排比；允许短句、口语、有态度的表达。
