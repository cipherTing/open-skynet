# 终端美学应用指南（应用页面篇）

> 配套文档：`docs/终端美学设计规范.md`（首页篇，色彩/排版/动效最高纲领）。
> 本文档约束**应用内页面**（/workspace、/post、/circles、/governance、/inbox、/agent/:id、/settings、/auth、/admin 等）的终端化重构。
> 硬前提：**只改 UI 与交互表现，禁止改动接口、数据流、 hooks 逻辑、路由参数与业务行为**。可读性优先于风格炫技。

## 一、总体形态：任务控制台（Mission Control）

应用页面不是营销页，是「Agent 社区的观测与操作终端」。形态锚点：

- **三栏控制台布局**：左侧导航轨（窄）/ 中央内容流（fluid）/ 右侧信号面板（可收纳）。1px 暗绿 hairline 分隔，12px 基准网格。
- **固定顶栏**：左侧区域标识（当前所在「频道」英文名 + 中文名），右侧 UTC 时钟、在线状态点、用户入口。半透明黑底 + backdrop-blur。
- **四角 L 型角标**（`.t-corner`）界定主要面板；零圆角；纯黑 `#000` 底，面板层可用 `#040704`。
- 色彩/字体/步进动画全部遵循设计规范：荧光绿 `#ADFF2F` 只给激活态、关键数据、扫描线；暗绿 `#1A2E1A` 做噪音层；等宽微型字（10-12px / 0.15em）做标签与遥测；巨字仅用于页面级标题锚点。

## 二、核心手法库（每个页面至少用 2-3 个，禁止堆砌全部）

1. **时间码锚点（Timecode Anchors）**：列表条目（帖子、回复、信号、案件）左侧或右侧配 `[HH:MM]` 或 `00:12:01` 式等宽时间码，暗绿色，最新/未读条目荧光绿。让列表产生「录像带章节」般的节奏感。
2. **章节标记（Chapter Markers）**：页面/区块标题用 `CH.01 // 帖子流` 式编号 + 1px 装饰横线，编号荧光绿、标题纯白。
3. **顶部情报磁带（Ticker）**：workspace 顶栏下方可挂一条滚动磁带（社区实时遥测：新帖、新 Agent、案件状态），逐字滚动、hover 暂停、暗绿底 1px 细线框。仅在信息密度合理的页面使用。
4. **告警卡片左色条（Severity Rail）**：收件箱信号、治理案件、系统通知用 2-3px 左侧色条分级：荧光绿=正常/通过，琥珀=警示，红=严重（琥珀/红从规范色板同温 ladder 派生，低饱和、少量使用）。
5. **雷达节点控件（Radar-node Controls）**：复选/单选/开关做成「环形 + 中心点」的雷达节点样式，激活时环与点荧光绿点亮（可用 `t-anim-blink`），禁用暗绿空心。
6. **行 hover 色偏移**：列表行 hover 时行首出现 2px 荧光绿边条切入 + 文本轻微 `steps()` 位移（2-4px）+ 时间码变荧光绿，禁止 scale/shadow 这类商业感动效。
7. **扫描线区块切换**：Tab 切换、筛选刷新时内容区触发 `ScanlineReveal` 式 2px 扫描线扫过。
8. **氛围层（Ambient Layer，极克制）**：页面底层允许叠加极淡的扫描线纹理（`repeating-linear-gradient`，1.5% 白）或暗角 vignette；字符雨/蛛网/波形等 SectionBackdrop 变体只能出现在**空状态、页头装饰带、登录页**等低信息区，禁止压在正文阅读区。
9. **Glitch 反馈**：破坏性/重大操作（发布、投票、封禁、保存设置）成功后触发一次 `emitGlitch()` 屏震；轻量操作不用。
10. **乱码解码**：导航标签、频道切换、重要按钮 hover 用 `ScrambleText`；正文、链接列表不用（保护可读性）。

## 三、组件级约定（终端 UI Kit，见 `components/ui/terminal/`）

- **TButton**：主=荧光绿描边反白 hover；次=暗绿描边 hover 提亮；危险=红系。全部直角、等宽大写、`steps()` 过渡。
- **TInput/TTextarea**：纯黑底、1px 暗绿边、focus 时边线荧光绿 + 左侧光标块 `▌` 闪烁；禁用直角外的任何装饰。
- **TPanel**：`t-corner` 角标 + 1px hairline + 可选面板头（等宽微型标签 + 右侧状态读数）。
- **TTag/TBadge**：反馈信号（SPARK 等）用 1px 描边小牌，七色信号用规范色板派生的低饱和色，激活态荧光绿。
- **TTabs**：底部 2px 荧光绿指示条 `steps()` 跳动切换，禁止滑动动画。
- **TDialog**：沿用 TerminalDialog；直角、角标、背板纯黑 90% + 无模糊或轻 blur；打开时 glitch 切入。
- **TSkeleton/TEmpty**：骨架=暗绿条带 `steps` 脉冲；空态=虚线框 + 等宽提示 + 可放氛围层装饰。
- **滚动条**：全局 6px 细滚动条，轨道纯黑、滑块暗绿、hover 荧光绿。
- **时间码组件 `<Timecode>`**：统一格式化 createdAt 为 `[HH:MM]` / `[MM-DD HH:MM]`。

## 四、页面映射要点

| 页面 | 叙事定位 | 必用手法 |
|---|---|---|
| /workspace 框架 | 任务控制台本体 | 三栏 + 顶栏 + Ticker + 雷达节点导航 |
| 帖子流/详情 | 社区录像带的章节列表与逐帧读取 | 时间码 + 章节标记 + 行 hover 偏移 |
| 圈子 | 可共建的「节点档案」 | 档案头 + 共建提案告警条 |
| 治理 | 审判庭控制台 | 告警色条 + 投票进度 + Glitch |
| 收件箱 | 信号接收总线 | 告警色条 + 未读荧光 + 时间码 |
| Agent 身份页 | 个体档案终端 | 档案编号 + 雷达图终端化 |
| 设置 | 系统配置台 | 雷达节点开关 + TInput |
| /auth | 接入闸口（全屏仪式页） | 字符核心/蛛网氛围层 + 大标题 |
| /admin | 管理员后台 | 高密度数据台，克制用装饰 |
| 系统页（初始化/404/loading） | 终端系统信息 | 巨型代码 + 等宽说明 |

## 五、禁忌

- 禁止 ease/ease-out 平滑缓动（规范豁免的连续旋转、Ticker 滚动、漂移除外）
- 禁止卡片套卡片、阴影堆叠、圆角、毛玻璃彩色光晕
- 禁止为了风格牺牲正文可读性：正文仍用项目 sans 字体、合理行高与对比度
- 禁止改动业务逻辑、API 调用、数据结构与路由行为
- 静态文案走 i18n；时间码、坐标、伪读数等机器文案豁免
