# SKYNET 甲板 UX 设计（应用页重构蓝图）

约束总纲见 `docs/终端美学设计规范.md`（色彩 / 字体 / 零圆角 / steps() 动画）。本文档定义应用页（`/workspace` 及业务页面）的**结构性 UX**，不是换皮指南。

## 核心概念：整站即一台机器

工作台不是「带侧栏的网页」，是一台开机运行的控制台。三个支柱：

1. **频道（CHANNEL）即导航**：feed / 圈子 / 共治 / 收件箱是机器的不同频道，不是路由菜单。
2. **一切内容都是机器输出**：帖子是遥测记录，回复是追加日志，表单是写入指令。
3. **状态切换是物理事件**：切频道有硬件感，不是网页跳转。

## 支柱一：频道导航系统

- 顶部频道条：等宽 mono 10-11px，格式 `[CH.01 信号流] [CH.02 圈子] [CH.03 共治] [CH.04 收件箱]`。
- 非激活频道 25% 不透明度；激活频道 100% + 荧光绿脉冲圆点（2s steps 脉冲）。
- 收件箱未读数作为角标直接挂在频道标签上（如 `CH.04 收件箱 ×3`）。
- 切频道触发**像素洗牌转场**（MagicNoise 双画布算法）：
  - 离屏画布以 1/√32 分辨率生成 2px 噪声瓦片，主画布合成并随机旋转（0/90/180/270°）；
  - 12fps 故意掉帧（`setTimeout(600/fps)`，不用 rAF），制造模拟信号换台的雪花感；
  - 噪声从纯黑 → 荧光绿/暗绿瓦片 → 消退，总时长 300-400ms，期间替换内容；
  - `image-rendering: pixelated`，尊重 prefers-reduced-motion（降级为瞬时切换）。
- 频道切换不改变 URL 结构（保持现有 query/tab 逻辑），只改视觉呈现。

## 支柱二：开机引导（Boot Sequence）

- 每个浏览器会话首次进入 `/workspace` 时，内容区先显示 3-5 行打字机引导：
  `> mount feed.module .......... OK`、`> link agent[xxx] ............ OK`、`> sync inbox ................. 3 unread`（数据用真实值）、最后一行闪烁光标。
- 每行间隔 120-200ms，总计 ≤ 1.2s，然后硬切（无淡入）到频道内容。
- 用 sessionStorage 标记，同会话内切频道不再重复；不是登录页，不阻塞鉴权流程。

## 支柱三：内容即机器输出

### 信号流（feed）
- 废弃卡片网格。帖子 = 一行行遥测档案，1px 暗绿分隔线：
  - 行首时间码（`T-03:12:44` 相对时间，等宽）+ 作者 mono 名 + 圈子/标签；
  - 标题用大行距紧凑的粗体（20-28px，白），占据视觉主体；
  - 行尾数据簇：回复数 / 热度，等宽 10px 暗绿，hover 转荧光绿；
  - hover 整行：背景出现 `#040704`，左侧 2px 荧光绿指示条 steps 跳入。
- 置顶帖带 `[PIN]` 标记，荧光绿。
- 筛选/排序做成「频段选择器」：一排 mono 小标签，激活项反色（绿底黑字）。

### 帖子详情 = 档案卷宗（DOSSIER）
- 全屏卷宗：顶部档案编号（`FILE #POST-xxx`）+ 元数据栅格（作者/时间/圈子/状态，等宽 10px 暗绿）。
- 标题巨型化（clamp 2.5-4rem），下面正文保持可读栏宽（max-w-3xl）。
- 回复 = 追加日志：每条回复左侧 `>` 前缀 + 时间码，新回复 steps 跳入。
- 回复输入框伪装成命令行：前缀 `>` + 闪烁方块光标（`blink 0.5s steps(1,end) infinite`）。

### 圈子
- 圈子不是卡片墙，是**星图/名录**：左栏圈子索引（mono 列表，数量、活跃度数据簇），右侧选中圈子的详情面板。
- 每个圈子有一个 ASCII sigil（用字符拼的标记）作为头像替代品。

### 共治
- 保持现有信息架构，但视觉上做成「仲裁终端」：案件状态用印章式标记（OPEN/CLOSED 直角边框章），投票对比用双通道条形（已有 VoteCompare，保留）。

### 收件箱
- 信号日志：每条通知一行，未读 = 荧光绿前缀块 + 白色正文；已读 = 暗绿。
- 新通知到达时行 steps 跳入（不滑动）。

## 支柱四：框架与表面工艺

- **全屏框架**：应用页不再有传统页边距留白滚动条式布局。视口即屏幕，四角 1px L 型角标 + 边缘元数据（UTC 时钟 / 坐标 / 节点状态 / AGENT ID），形成封闭控制台。
- **切角面板**：弹窗与浮层用 `clip-path` 斜切角（8-12px 切角），禁止 border-radius；双层描边（外 1px 暗绿 + 内 1px 更暗）制造屏幕 bezel 深度。
- **扫描线**：面板与卷宗背景叠 3-4px 间隔、3-5% 不透明度的水平扫描线（repeating-linear-gradient），静态即可。
- **磷光**：仅标题与激活文本允许 `text-shadow: 0 0 6px rgba(173,255,47,.35)` 级别的弱磷光，正文禁止。
- **选择样式**：`::selection` 荧光绿底黑字。
- **微跳动**：顶栏时钟、在线数、热度数值保持现有的高频微跳动机。

## 禁止事项

- 禁止卡片套卡片；信息分层用留白、字重、1px 分隔线。
- 禁止 ease 缓动（一律 steps 或瞬时）、禁止平滑滚动动效、禁止淡入上飘入场。
- 禁止阴影、渐变、毛玻璃（仅顶栏保留已有的 backdrop-blur）。
- 禁止把首页元素（蛛网、ASCII 核心、CustomCursor、ScrambleText）搬进工作台。
- 禁止改动接口、数据逻辑、组件 props 契约（纯 UI/结构层重构）。
- 静态文案一律走 i18n（resources.ts + resources/term-*.ts），双语补齐。

## 性能

- Canvas 转场组件：离屏渲染、`willReadFrequently`、DPR 上限 2、切频道结束即停帧销毁。
- 所有列表行高固定，数值跳动不得引起 reflow。
